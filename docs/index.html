Lispish
v0.1.1

Welcome to Lispish!


Lispish is a Lisp dialect made to work well in a browser or Node environment.

Quite a lot in Lispish is not what you're used to if you've done some Lisp before.

  • t
    and 
    nil
    are gone. Instead there are four new symbols: 
    true
    , 
    false
    , 
    null
    and 
    undefined
    .
  • Only one sequence type exists: 
    list
    . And its undelaying data structure is a javascript array.
  • No quotes! 
    '(1 2 3)
    is no more... Use 
    (list 1 2 3)
    instead.
  • No macros.
  • No keyword symbols e.g. 
    :foo
    .
  • No tail call optimization (yet).
  • No dotted pairs.
  • 100% test coverage

Have a look at the list of functions to the left. These are what is available in terms of special- and normal expressions.

For more instruction on how to install and use Lispish as a cli or a typescript lib, checkout https://github.com/mojir/lispish

Happy coding!

▶ (first (list "Albert" "Mojir" 160 (list 1 2))) => "Albert"
▶ (first (list)) => undefined
last

Returns the last element of list. If list is empty, undefined is returned.

last list => any
list: list
▶ (last (list "Albert" "Mojir" 160 (list 1 2))) => [1,2]
▶ (last (list)) => undefined
rest

Returns a new list with all but the first element from list. If list has less than two elements, an empty list is returned.

rest list => list
list: list
▶ (rest (list "Albert" "Mojir" 160 (list 1 2))) => ["Mojir",160,[1,2]]
▶ (rest (list "Albert")) => []
▶ (rest (list)) => []
cons

Constructs a new list with element as first element and rest as the rest.

cons list => list
list: list
▶ (cons "Hi" (list "Albert" "Mojir" 160 (list 1 2))) => ["Hi","Albert","Mojir",160,[1,2]]
▶ (cons "Hi" (list)) => ["Hi"]
▶ (cons "Hi") => Error!
object

Constructs a new object. Object members are created from key - value pairs. Requires an even number of arguments.

object [key value](zero or more) => object
[key value]: [string any]
▶ (object) => {}
▶ (object "x" 10 "y" true "z" "A string") => {"x":10,"y":true,"z":"A string"}
keys

Returns list of all keys in object.

keys object => list
object: object
▶ (keys (object)) => []
▶ (keys (object "x" 10 "y" true "z" "A string")) => ["x","y","z"]
values

Returns list of all values in object.

values object => list
object: object
▶ (values (object)) => []
▶ (values (object "x" 10 "y" true "z" "A string")) => [10,true,"A string"]
entries

Returns nested list of all key - value pairs in object.

entries object => list of [key value] - paris
object: object
▶ (entries (object)) => []
▶ (entries (object "x" 10 "y" true "z" "A string")) => [["x",10],["y",true],["z","A string"]]

merge object(one or many) => object

setq variable value => any
variable: name
value: any

let

Special Expression

Binds local variables. The variables lives only within the body. It returns evaluation of the last expression in the body.

let bindings body => any
bindings: bindings
body: lisp expressions
▶ (let ((a (+ 1 2 3 4)) (b (* 1 2 3 4))) (write a b)) => 24
lambda

Special Expression

Creates a function. When called, evaluation of the last expression in the body is returned.

lambda arguments body => function
arguments: arguments
body: lisp expressions
▶ (lambda (a b) (sqrt (+ (* a a) (* b b)))) => <FUNCTION λ (a b)>
▶ ((lambda (a b) (sqrt (+ (* a a) (* b b)))) 3 4) => 5
defun

Special Expression

Creates a named global function. When called, evaluation of the last expression in the body is returned.

defun name arguments body => function
name: name
arguments: arguments
body: lisp expressions
▶ (defun hyp (a b) (sqrt (+ (* a a) (* b b)))) #'hyp => <FUNCTION hyp (a b)>
▶ (defun hyp (a b) (sqrt (+ (* a a) (* b b)))) (hyp 3 4) => 5
function

Special Expression

Accessing namespace of functions. Shortform #' is equivalent.

function name arguments body => function
name: name
arguments: arguments
body: lisp expressions
▶ (function +) => <BUILTIN FUNCTION +>
▶ #'+ => <BUILTIN FUNCTION +>
▶ (defun hyp (a b) (sqrt (+ (* a a) (* b b)))) (function hyp) => <FUNCTION hyp (a b)>
if

Special Expression

Either then or else branch is taken. Then branch is selected when test result is truthy.

if test then else => any
test: any
then: any
else: any
▶ (if true (write "TRUE") (write "FALSE")) => "TRUE"
▶ (if false (write "TRUE") (write "FALSE")) => "FALSE"
cond

Special Expression

Used for branching. Variants are tested sequentially from the top. I no branch is tested truthy, undefined is returned.

cond variants => any
variants: variants
▶ (cond (false (write "FALSE")) (null (write "NULL")) (true (write "TRUE"))) => "TRUE"
▶ (cond (false (write "FALSE")) (null (write "NULL"))) => undefined
regexp

Creates a RegExp from regexp-string and flags.

regexp regexp-string flags => RegExp
regexp-string: string
flags: string
▶ (regexp "^\s*(.*)$") => /^\s*(.*)$/
▶ (regexp "albert" "i") => /albert/i
match

Matches string against regexp. If string matches, a match-list is returned, otherwise undefined.

match regexp string => list
regexp: RegExp
string: string
▶ (match (regexp "^\s*(.*)$") "  A string") => ["  A string","A string"]
▶ (match (regexp "albert" "i") "My name is Albert") => ["Albert"]
▶ (match (regexp "albert" "i") "My name is Ben") => undefined
